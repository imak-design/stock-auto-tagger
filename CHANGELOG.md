# 変更履歴（CHANGELOG）

Stock Auto Tagger の主な変更点をまとめています。**新しい変更ほど上**に記載します。

各サイト（Adobe Stock / Shutterstock / Pixta）はページの作りが予告なく変わることがあり、その都度アップロード・提出処理を修正しています。アップデートがうまく動かないときは、まずこのページで最近の修正内容をご確認ください。

> 記号の意味: 🆕 追加機能 / 🛠 修正 / 🔧 変更・改善 / 🔒 セキュリティ / 📖 ドキュメント

---

## 2026-07-20

- 🛠 **Adobe Stock で提出に失敗しても通知されずに処理が進んでしまう残りのケースを修正（レビュー指摘の反映）**
  審査登録ボタンが見つからない等の理由で提出できなかった場合に「提出完了 0件」とだけ表示されて気づけないケースが残っていたため、人間確認画面と同じ「要手動対応」ポップアップでお知らせするようにしました。メタデータCSVの適用が途中で止まった場合もエラーとして通知します。
- 🔧 **人間確認画面の誤検出を防止**
  ページに常駐する見えない確認部品まで検出して、正常に提出できているのに「要手動対応」と表示される可能性があったため、実際に画面に表示されている確認画面だけを検出するようにしました。
- 🔧 **提出完了の確認待ちを最大約2分→約6分に延長**
  ファイル数が多いときに、Adobe 側の反映が遅いだけで「未提出」と判定されるのを防ぎます。時間切れの場合の案内文にも「反映が遅れているだけの可能性がある」ことを明記しました。
- 🔧 テストモードでは完了表示を「メタデータ適用 N件」に変更（実際には提出していないのに「提出完了 N件」と表示されていました）。
- 🛠 **Adobe Stock が実際には審査提出できていないのに「提出完了」と表示されていた問題を修正**
  Adobe は人間かどうかを確認するため、提出の直前に画像選択（猫の画像を選ぶなど）や単語入力の確認画面をランダムに表示することがあります。これが出るとファイルはアップロードされても審査提出は完了しませんが、これまでは完了として報告し、そのままファイル移動まで進んでしまっていました。提出後に実際に受け付けられたかを確認し、できていない場合は件数を「0件」として正しく報告します。
- 🆕 **人間確認画面が出たらポップアップでお知らせするようにしました**
  自動実行中は別の作業をしていて気づけないため、最前面のポップアップで通知します。ブラウザは開いたままにするので、そのまま画像選択などに対応して手動で審査提出を完了してください。
  - **Adobe 単体で実行した場合**: その場でポップアップを表示し、手動対応待ちの状態を保ちます
  - **全工程を実行した場合**: いったんログに記録して Shutterstock・Pixta の処理を続行し、最後にまとめてポップアップでお知らせします（ファイルは Adobe にアップロード済みのため、ファイル移動は通常どおり行われます）
- 🔧 **単語入力の確認画面へ自動で回答するのをやめました**
  これまでは画像の内容と関係のない固定の単語を入力して送信していました。誤った回答を繰り返すとアカウントの評価に影響する可能性があるため、自動回答は廃止し、人間に引き継ぐ方式に統一しました。
- 📖 マニュアル（工程2・トラブルシューティング）と README に、Adobe の人間確認画面が出たときの動作と対処を追記しました。

## 2026-07-02

- 🔒 **セキュリティ強化（外部レビュー結果の反映）**
  - Pixta 動画のタグ入力（クリップボード経由）で、AI が生成したタグをコマンドに直接埋め込まない方式に変更（コマンドインジェクション対策・日本語タグの文字化け防止）
  - Gemini API キーを URL ではなくヘッダーで送信するように変更（通信ログ等へのキー露出を防止）
  - 工程0リネームで AI の応答をファイル名に使う前に英数字のみへ正規化（想定外の文字混入対策）
  - 画像メタデータ（XMP）埋め込み時にタイトル・キーワードの特殊文字を正しくエスケープ
  - 起動時ログの API キー表示を「読み込み済み」のみに変更（一部マスク表示を廃止）
- 🛠 **安定性の改善**
  - テストモードの全自動実行を繰り返すとブラウザ制御プロセスが溜まっていく問題を修正（再実行時・アプリ終了時に解放）
  - Gemini へのファイルアップロードが一時的な通信エラーで即失敗していたのを、自動リトライするように改善
- 🔧 **整理**
  - 使われていなかった FTPS アップロード処理を削除（実際のアップロードはブラウザ自動化のみ）
  - `requirements.txt` の各パッケージにバージョン範囲を指定（インストール時の予期しない非互換を防止）
- 📖 README のフォルダ構成説明を実態に合わせて修正（`variation/01〜10` は手動作成であることを明記、`config.json` サンプルに `test_mode` を追加）

## 2026-06-22

- 🛠 **Shutterstock にアップロードできない問題を修正**
  Shutterstock 出品ページの仕様変更に追従しました。次の不具合を解消しています。
  - 「アップロードボタンが見つかりません」で止まる → 新しいファイル選択方式（ドロップゾーン）に対応
  - 審査提出が「提出ボタンが見つかりません」で止まる → 提出ボタンが素材選択中のみ表示される新仕様に対応
  - メタデータ確認の途中でページが再読み込みされ、選択が外れて提出できなくなる問題を解消
