Stock Auto Tagger

GUI 操作マニュアル — ボタンを押すだけで 3 サイトへ一括アップロード

このマニュアルは 起動後の GUI 操作専用です。Python / Git のインストール、依存パッケージ、API キー、各サイトへのログインなど セットアップ手順は README.md を参照してください。
⚠️ アプリ起動前に最低 1 サイトへのログインが必要です。起動時に enabled_sites のセッションをチェックし、1 つも保存されていない場合はエラーで即終了、一部未保存なら警告ダイアログを表示して起動継続（未保存サイトはアップロード時に自動スキップ）します。ログインは python program/<site>_login.py を CLI で実行してください（GUI 内からのログインは Pixta reCAPTCHA 対策のため非対応）。詳細は README §6。

1. はじめに

ストック素材（画像・動画・ベクター EPS）を Adobe Stock / Shutterstock / Pixta の3サイトへ一括登録する自動化ツールです。Gemini AI でタイトル・キーワード・カテゴリを生成し、Playwright でブラウザを自動操作してアップロード〜審査提出まで全自動で実行します。

対応サイトとファイル形式

サイト画像動画ベクター
Adobe StockJPG / PNGMP4 / MOVEPS
ShutterstockJPGMP4 / MOVEPS
PixtaJPG / PNGMP4 / MOVZIP (EPS+PNG)

※ Shutterstock は PNG 非対応のため、JPG と動画のみアップロードされます。
※ Shutterstock・Pixta は AI 生成素材の受付を停止中のため、AI 素材はこれらのサイトから自動除外されます。

各部位の説明

#項目説明
1素材フォルダタグ生成とアップロードの対象ファイルを置く場所。デフォルトはプロジェクト内の input/。
2バリエーションフォルダ工程0で色違いリネームする素材の置き場。空欄でも OK（おまけ機能）。詳細は §3。
3アップロード先有効にするサイトのチェックボックス。OFF のサイトは全工程でスキップされます。
4テストモードON にすると各サイトで「審査申請ボタンの手前」で停止。詳細は §6。
5概算リクエスト数これから押す処理で消費する Gemini API のリクエスト数目安。次項参照。
6全自動ボタン（上段）最後まで一気通貫で実行。詳細は §4。
7個別実行ボタン（下段）1工程ずつ実行。部分的な再処理に便利。
8ログ・進捗バー実行中の進行状況をリアルタイム表示。

概算リクエスト数と無料枠について

モード行の右側に、次に押す処理で消費する Gemini API リクエスト数の概算が表示されます。Gemini API の無料枠には1日の上限があり、バッチ処理（画像最大10枚＝1リクエスト）で消費を抑えていますが、目安は把握しておきましょう。

1日に処理できる素材数の目安（無料枠）

処理内容1日あたり
画像のタグ生成（工程1）約 100〜200 枚（10枚バッチで処理）
動画のタグ生成（工程1）約 20 本（1本 = 1リクエスト）
バリエーションリネーム（工程0）約 20 セット（1セット = 1リクエスト）
リネーム（工程0）とタグ生成（工程1）は別のモデルを使うため、上限枠も別々にカウントされます。
  • 工程0 リネーム → gemini-2.5-flash-lite の枠
  • 工程1 タグ生成 → gemini-3-flash-preview の枠

つまり 「リネームの枠を使い切ったらタグ生成もできなくなる」ということはありません。逆も同じく、タグ生成の枠を使い切っても翌日リセットまでリネームは動きます。

※ 枠が独立しているだけで、処理の中身は振り替えできません（例: リネーム用の枠を使ってタグ生成はしません）。

※ 無料 API の仕様・上限は変更される場合があります。最新値は Google AI Studio のレート制限ページ を確認してください。上限は日本時間 17:00 頃にリセットされます。

アップロード前の自動バリデーション

アップロードボタンを押すと、各サイトの制限を自動チェックします：

  • Adobe: 画像 45MB まで
  • Shutterstock: 50MB まで
  • Pixta: 画像 50MB まで（PNG は 30MB まで）、1回 50 枚まで

制限超過ファイルは自動的に除外されてログに表示されます。

3. 素材フォルダの使い分け

input/                        ← プロジェクト内に同梱済み
├── image1.jpg            ← 通常素材（直下）
├── video1.mp4
├── AI/                   ← AI 生成素材
│   └── ai_image.png
├── Photo/                ← 写真素材
│   └── DSC01234.jpg
├── Vector/               ← ベクター素材
│   └── 260402_icon/      ← 素材ごとにサブフォルダ
│       ├── 260402_icon.eps
│       └── 260402_icon.png
└── csv_output/           ← CSV 自動出力（自動作成）

variation/                    ← おまけ機能（工程0用）
└── 01/ 〜 10/             ← 画像バリエーション
    └── movie/01/ 〜 10/   ← 動画バリエーション

input/ 直下 — 通常のイラスト・動画

サブフォルダに入れなかった素材は、すべて通常のイラストとして処理されます。

  • Adobe: イラストカテゴリ
  • Shutterstock: 通常
  • Pixta: イラストページ

input/AI/ — AI 生成画像

AI で生成した画像は必ずここに入れてください。

  • Adobe: AI 生成の申告が自動 ON
  • Shutterstock・Pixta: 自動的に除外（AI 素材受付停止のため）

input/Photo/ — 写真素材

  • Adobe: 写真カテゴリで登録
  • Shutterstock: 通常
  • Pixta: 写真ページで登録

input/Vector/ — ベクター素材（EPS + PNG）

素材ごとにサブフォルダを作り、EPS と PNG をセットで入れてください。

Vector/
└── 260402_icon/
    ├── 260402_icon.eps   ← ベクター本体
    └── 260402_icon.png   ← プレビュー用
  • Adobe・Shutterstock: EPS をそのままアップロード
  • Pixta: EPS + PNG を自動で ZIP 化してアップロード

variation/ — 【おまけ】色違いリネーム機能

必須ではありません。色違いバリエーションを大量に作るときだけ使う、工程0専用のおまけ機能です。

何をする機能？

サブフォルダにバラバラのファイル名で素材を入れておくと、Gemini が画像を解析して 日付_キーワード_色.拡張子 の形式に自動リネーム → input/ 直下へ移動します。

フォルダ構造

variation/
├── 01/                   ← 1セット目（色違い群）
│   ├── aaa.png           ← → 260415_crown_blue.png 等に自動リネーム
│   ├── bbb.png
│   └── ccc.png
├── 02/
│   └── ...
└── movie/
    └── 01/               ← 動画セットも同様の構造
色違いを想定した機能です。デフォルトプロンプトは「色名を判定する」ように書かれているため、ポーズ違い・形違いなど色以外のバリエーションを入れると命名が破綻します。そういう素材は variation/ ではなく input/ 直下に入れてください。
黒背景の素材推奨。色判定の精度を上げるため、デフォルトプロンプトは黒背景前提で書かれています。

セーフティネット（元素材を残します）

リネーム対象の素材は 元ファイルを削除せずコピーを作成し、元ファイルはゴミ箱へ送られます。AI の色判定がおかしかった場合でも、ゴミ箱から復元できるので安心して試せます。

4. ボタンごとの処理内容

全自動ボタン（上段）

押すだけで最後まで一気通貫で実行。

⚡ 【工程0→全自動】リネーム → タグ → アップロード ▶ 【工程1→全自動】タグ生成 → アップロード
ボタン処理の流れ
工程0→全自動 バリエーションリネーム → タグ生成 → 3サイトアップロード → ファイル移動
工程1→全自動 タグ生成 → 3サイトアップロード → ファイル移動

個別実行ボタン（下段）

1工程ずつ実行したいときに使います。各工程を「入力 → 処理 → 出力」で整理しました。

⚡ 工程0 — バリエーションリネーム

入力variation/01/ 〜 variation/10/、variation/movie/01/ 〜 の素材
処理Gemini Flash-Lite でキーワードと色を1回のAPIコールで判定
出力日付_キーワード_色.ext 形式にリネームして input/ 直下へ移動（元ファイルはゴミ箱）

▶ 工程1 — タグ生成 & CSV 出力

入力input/ 内の全素材（直下・AI・Photo・Vector）
処理Gemini 3 Flash でバッチ解析（画像最大10枚/リクエスト）。動画は個別処理
出力input/csv_output/ に Adobe / Shutterstock 用 CSV。Pixta 用は IPTC/XMP を画像に直接埋め込み。動画解析結果は video_metadata.json に保存

☁ 工程2 — Adobe Stock アップロード

入力input/ 全素材 + 工程1 の CSV
処理ブラウザ自動アップロード → CSV 適用 → 各ファイルのコンテンツタイプ設定（AI 素材は AI チェック ON）→ 審査提出
出力Adobe Stock の審査キューに登録

☁ 工程3 — Shutterstock アップロード

入力JPG + 動画 + EPS（PNG・AI 素材は除外）+ 工程1 の CSV
処理ブラウザ自動アップロード → CSV 適用 → 画像タブ・動画タブの順に提出
出力Shutterstock の審査キューに登録

🌸 工程4 — Pixta 3フェーズアップロード

入力画像 + 動画 + Vector（ZIP 化）+ 工程1 の埋込メタデータ。AI 素材は除外
処理フェーズ1: イラスト + ベクター → フェーズ2: 動画（工程1の video_metadata.json を再利用）→ フェーズ3: 写真
出力Pixta の審査キューに登録

📦 工程5 — ファイル移動

入力処理済み素材
処理アーカイブフォルダへ移動（テストモード中はスキップ）
出力input/ がクリアされ、次の素材を受け入れ可能に

5. 使い方シナリオ

シナリオ A: 通常の一括登録（基本の使い方）

  1. input/ に JPG / PNG / MP4 を入れる（AI 生成は AI/、写真は Photo/、ベクターは Vector/素材名/ へ）
  2. アプリ起動 → サイトのチェックを確認
  3. 【工程1→全自動】をクリック
  4. タグ生成 → 3サイトアップロード → ファイル移動まで全自動

シナリオ B: バリエーション（色違い）込みで登録

  1. variation/01/〜10/ に色違いセットを入れる（1セット = 同一モチーフの色違い群）
  2. 通常素材があれば input/ にも入れておく
  3. 【工程0→全自動】をクリック
  4. リネーム → 全素材のタグ生成 → アップロード → ファイル移動まで自動

※ 色違い以外（ポーズ違い等）は variation/ に入れず input/ 直下へ。

シナリオ C: 特定のサイトだけに登録したい

  1. アップロード先のチェックを 登録したいサイトだけ ON、他は OFF
  2. あとはシナリオ A / B と同じ操作で OK

例: Adobe だけ登録したいなら Shutterstock と Pixta のチェックを外す → 全自動を押す。

シナリオ D: 途中でエラー → 部分的にやり直したい

個別実行ボタンを使います。前工程の成果物（CSV / メタデータ）は残っているので再利用できます。

  • Adobe だけ再実行 → 【工程2】Adobe をクリック（タグ生成は再実行不要）
  • Pixta だけ再実行 → 【工程4】Pixta をクリック
  • タグ内容を手動修正してから再アップ → CSV を編集 → 【工程2〜4】を順番にクリック
  • ファイル移動だけ実行 → 【工程5】ファイル移動

6. テストモード

GUI の「テストモード」チェックを ON にすると、各サイトで 審査申請ボタンの手前で処理が停止します。タイトル・タグを手動で確認・修正できます。

サイト停止位置
Adobe Stock一括アップロード → CSV 適用 → コンテンツタイプ設定 → 全選択完了 → 「審査に登録」ボタン手前
Shutterstock一括アップロード → CSV 適用完了 → 提出ボタン手前
Pixta 画像フェーズごとにアップロード → 全選択完了 → 「選択した作品を登録」ボタン手前
Pixta 動画タイトル / タグ入力完了 → 審査申請の手前
処理完了 → ブラウザが開いたまま停止
↓
ブラウザ上でタイトル・タグを確認 / 修正
↓
ブラウザを閉じる → 自動的に次のサイトへ
  • テストモード時はファイル移動もスキップ → 同じ素材で繰り返しテスト可能
  • 全自動ボタン: ブラウザを閉じずに次の工程へ自動で進み、全工程完了後にポップアップで通知
  • 個別ボタン: ブラウザを閉じると次の処理に進みます。ブラウザ待機中はポップアップで通知されます
  • ブラウザ内でページ遷移（審査申請ページへ移動など）しても問題ありません

7. トラブルシューティング

症状原因対処
起動時に「ログインセッションがありません」エラー 有効サイトのセッションが 1 つも保存されていない ダイアログに表示されたいずれかのサイトで python program/<site>_login.py を CLI 実行してからアプリ再起動
起動時に「一部サイトのログインセッションが未保存」警告 一部サイトだけセッション保存済み 未保存サイトはアップロードでスキップされます。利用したい場合はダイアログ表示のコマンドでログイン
Session expired / 途中で停止 サイトのログインセッション切れ 該当の *_login.py を再実行してセッションを更新
Gemini 429 エラー API レート制限 / 無料枠超過 自動リトライで回復待ち。1日の上限に達した場合は日本時間 17:00 頃にリセット
Pixta でタグが 0 個 Gemini レスポンス異常 ログを確認し、プロンプト（§8）を調整
API キーが見つからない .env ファイルが未作成 README.md の「Gemini API キーの設定」を参照
ブラウザが自動で閉じない
（テストモード）		 正常動作 確認後、手動でブラウザを閉じてください
CSV 適用後にタグが空 ポータル反映遅延 自動リトライで待機。解消しない場合は手動でリロード
ファイルが除外された サイトのサイズ制限超過 ログでファイル名を確認 → リサイズ or 別途手動アップロード（Adobe 45MB / Pixta PNG 30MB / Shutterstock 50MB）
バリエーションのリネームが変 色違い以外が混ざっている / 背景が白 variation/ は色違い専用。色判定が不適切なら元ファイルをゴミ箱から復元

8. カスタマイズ

プロンプトの編集

program/prompts/ 内のテキストファイルを編集するだけで、AI のタグ生成ルールを変更できます。Python コードを触る必要はありません。

ファイル用途
program/prompts/image_prompt.txt画像用プロンプト（工程1）
program/prompts/video_prompt.txt動画用プロンプト（工程1）

Claude でカスタマイズ

Claude（AI アシスタント）にコードを見せて、やりたいことを伝えるだけでカスタマイズできます：

  • 「Adobe Stock だけ使いたいので、他のサイトの処理を削除して」
  • 「動画は扱わないので、画像だけのシンプル版にして」
  • 「タグの生成ルールを変えたい」
